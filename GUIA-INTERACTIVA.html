20 minutos · El espejo

Antes del nodo existe una decisión.

Casi todos los equipos ya tienen automatizaciones: una hoja que alguien actualiza, una reunión que revela el problema tarde, un mensaje que se pierde. El reto es convertirlas en evidencia que una persona pueda usar.

PREGUNTA 01

¿Cuál fuente manda?

Proyecto, calendario y operación suelen vivir en herramientas distintas. No hay una “verdad única” por defecto.

PREGUNTA 02

¿Qué es riesgo?

Un estado no basta: riesgo combina fecha, prioridad, dependencia y responsabilidad explícita.

PREGUNTA 03

¿Quién puede decidir?

La automatización prepara contexto. Un humano aprueba prioridades, presupuestos, mensajes externos y cambios de estado.

Los cinco conceptos n8n

  1. Trigger: cuándo empieza: manual, horario, webhook o evento.
  2. Nodo: una operación: consultar, transformar, decidir o notificar.
  3. Item: un registro JSON que viaja por el flujo.
  4. Credencial: permiso guardado localmente, no texto pegado en el workflow.
  5. Ejecución: evidencia de qué ocurrió y con qué datos.

30 minutos · Fallas reales de diseño

Lo que parece automático puede ser opaco.

La mayoría de los incidentes no ocurren porque un nodo “falló”; ocurren porque nadie definió qué significa una alerta, qué datos son confiables o quién debía revisar antes de enviar.

BRECHA 01

Credenciales en el JSON

Una API key copiada en un nodo termina en historial, capturas o Git. Las credenciales se crean dentro de n8n y los exports no incluyen secretos.

BRECHA 02

IA como calculadora

Pedir a un modelo que “decida prioridades” sin reglas convierte una operación medible en opinión no trazable.

BRECHA 03

Salida sin compuerta

Un correo o cambio de estado externo puede ser irreversible. La aprobación humana está en el diseño, no en una nota al final.

CASOS · 03elige una respuesta

¿Cuál es la corrección de arquitectura?

Escenario: el informe dice que “Migración CRM está al 90%”, aunque el snapshot no contiene progreso. El flujo envía el correo automáticamente a dirección.

Detener el flujo es una capacidad: “Send and Wait for Approval”, un formulario, una tarea o una rama de revisión son mejores que publicar un resultado que nadie verificó.

30 minutos · Receta reproducible

Ahora llévalo a n8n local.

No necesitas comprar nada para entender la arquitectura. El workflow 01 funciona sin credenciales. Para fuentes reales, cada persona conecta sus propias cuentas con permisos mínimos.

Ruta local

  1. Instala Docker Desktop y copia .env.example a .env.
  2. Define una clave de cifrado larga; nunca subas .env.
  3. Ejecuta docker compose up -d y abre localhost:5678.
  4. Importa 01-semilla-demostracion.json y ejecútalo.
  5. Importa 02-torre-de-control.json para conocer el contrato.

Ruta de una fuente real

  1. Crea la credencial en n8n: OAuth o API key con alcance mínimo.
  2. Consulta sólo el conjunto necesario: eventos próximos, issues abiertos o filas activas.
  3. Agrega Edit Fields o Code para mapear al contrato.
  4. Envía el lote al webhook de la Torre y revisa la ejecución.
  5. Sólo después escribe el dashboard o genera el informe.
